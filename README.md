# CAMPON - backend

> "캠핑장을 많이 보여주는 서비스가 아니라, 사용자가 실제로 실패 없이 갈 수 있는 캠핑을 추천한다."

초보 캠퍼가 조건을 입력하면 실제로 갈 수 있는 캠핑장을 추천해주는 캠핑 준비 지원 서비스의 백엔드 저장소입니다. 아래 문제 정의·타깃 사용자는 서비스가 지향하는 전체 방향이며, 현재 백엔드에 구현된 범위는 "핵심 기능" 항목을 기준으로 확인해 주세요.

## 문제 정의

캠핑은 인기가 높은 여행·휴식 활동이지만, 초보 캠퍼나 가족 단위 이용자는 캠핑장 선택부터 장비 준비, 이동 가능성 확인, 날씨 대응까지 여러 요소를 스스로 판단해야 합니다.

- 내 상황에 맞는 캠핑장을 찾기 어렵다.
- 차량이 없을 경우 실제로 이동 가능한 캠핑장인지 판단하기 어렵다.
- 어떤 장비가 필요한지 몰라 과소비하거나 준비물을 빠뜨린다.
- 화장실, 샤워실, 전기, 가족 편의시설 등 이용 편의성을 비교하기 어렵다.
- 날씨 변화(강풍, 우천, 한파, 폭염 등) 리스크를 사전에 판단하기 어렵다.
- 캠핑 전 준비 사항을 체계적으로 관리하기 어렵다.

기존 캠핑 정보 서비스는 대부분 캠핑장 검색·예약·리뷰 중심으로 구성되어 있어, 사용자의 날짜·인원·이동수단·숙련도·장비 보유 상태를 종합적으로 고려한 맞춤형 준비 지원은 부족합니다.

## 타깃 사용자

**핵심 사용자**
- 캠핑 경험이 적은 초보 캠퍼
- 차 없이 캠핑을 가고 싶은 뚜벅이 캠퍼
- 차박을 하고 싶은 사람
- 가족 단위 캠핑객
- 장비를 아직 충분히 갖추지 못한 사용자
- 캠핑장을 고르는 데 시간이 오래 걸리는 사용자

**보조 사용자**
- 캠핑장을 운영하는 사업자
- 지역 관광 활성화를 원하는 지자체
- 캠핑 장비 대여·판매 업체
- 캠핑장 주변 관광지·상권

## 핵심 기능

- **캠핑장 추천** — 거리, 선호 시설 일치도, 장비 대여 가능 여부, 차량/트레일러/카라반 사이트 적합도를 가중치 기반으로 점수화하여 추천 (`CampsiteRecommendService`, `CampsiteScoreCalculator`)
- **이동 수단 적합도 체크** — 차량·트레일러·카라반 여부에 맞는 사이트인지 확인
- **시설 점수화** — 선호 시설(샤워장, 화장실, 전기 등) 일치도와 보험·소방 안전 정보를 점수화
- **장비 대여 가능 분석** — 기본 장비 보유 여부와 캠핑장의 대여 장비를 비교해 부족한 장비의 대여 가능 여부를 점수화
- **커뮤니티** — 게시글 등록·조회·수정·삭제 (작성자 권한 검증 포함)
- **인증** — Google/Apple/Kakao OAuth2 로그인, JWT 발급·검증

## 차별점

기존 서비스가 "어떤 캠핑장이 있는가"를 보여주는 데 집중했다면, 본 서비스는 사용자 조건(거리, 시설, 장비, 이동수단)에 맞춰 캠핑장 후보를 서버 로직으로 직접 점수화하여 추천합니다.

- 개인 조건(거리·시설·장비·이동수단) 기반 캠핑장 추천 점수화
- 차량/트레일러/카라반 사이트 적합도 판단
- 보유 장비 대비 대여 가능 장비 기반 부족 장비 분석
- 화장실·샤워실·전기 등 선호 시설 일치도 및 안전 정보 점수화

추천 점수는 LLM이 아니라 서버의 `ScorePolicy` 구현체(거리, 시설, 장비, 이동수단, 안전 등)에서 가중치를 적용해 계산됩니다.

## 데이터 활용

- 캠핑장 위치 및 상세 정보 — 한국관광공사 고캠핑 정보 조회서비스(TourAPI) 연동 완료 (`TourApiClient`)
- Gemini API 연동 코드가 존재하지만, 현재는 테스트용 엔드포인트에서만 호출되며 추천/정규화 로직에는 아직 사용되지 않습니다.

## 기술 스택

| 구분 | 기술 |
| --- | --- |
| Backend | Spring Boot 4.1 (Java 25) |
| Database | MariaDB |
| Security/Auth | Spring Security, JWT, Google/Apple/Kakao OAuth2 |
| 외부 API | TourAPI (한국관광공사 고캠핑) |
| API 문서 | springdoc-openapi (Swagger UI) |
| 통신 | Spring WebFlux (WebClient) |
| LLM | Gemini API 연동 (현재 테스트 엔드포인트 전용, 프로덕션 로직 미연동) |

## 시작하기

```bash
./gradlew build
./gradlew bootRun
```

## 프로젝트 구조

```
src/main/java/com/seohamin/campon
├── domain
│   ├── auth        # 로그인/OAuth(Google, Apple, Kakao), JWT 인증
│   ├── user         # 회원 정보 및 관리
│   ├── campsite      # 캠핑장 추천/조회
│   └── post         # 커뮤니티 게시글
└── global           # 공통 설정, 예외 처리 등
```
